---
title: "Authentication"
description: "How to obtain a Bearer token and authenticate your requests to the Oriyn API."
---

The Oriyn API uses Bearer token authentication. You must include a valid token on every request except the health check endpoint.

## Getting a token

Oriyn uses JWTs issued through a browser-based OAuth flow. You can obtain a token in two ways:

**From the CLI:** Run the following command and complete the browser login. The CLI stores your token in your OS keychain and uses it automatically for all CLI commands.

```bash
oriyn login
```

**From the web app:** Log in at [app.oriyn.ai](https://app.oriyn.ai). Your session token can be retrieved from your browser's local session for use in direct API calls during development.

## Using your token

Pass your token in the `Authorization` header on every API request:

```
Authorization: Bearer <your-token>
```

<CodeGroup>

```bash cURL
curl https://api.oriyn.ai/v1/me \
  --header "Authorization: Bearer <your-token>"
```

</CodeGroup>

```json Response
{
  "user_id": "a1b2c3d4-e5f6-7890-abcd-ef1234567890",
  "email": "you@example.com"
}
```

## Token expiry and refresh

Tokens are JWTs and expire automatically. When your token expires, requests will return `401 Unauthorized`.

To get a new token:

- **Web app:** Log in again at [app.oriyn.ai](https://app.oriyn.ai).
- **CLI:** Run `oriyn login` again.

<Note>
Store your token securely. Do not commit it to version control or expose it in client-side code. Use environment variables or a secrets manager in production.
</Note>

## Auth error responses

| Status | Meaning |
|---|---|
| `401 Unauthorized` | Your token is missing, malformed, or expired. Check that you're passing the `Authorization: Bearer <token>` header correctly, and refresh your token if needed. |
| `403 Forbidden` | Your token is valid, but you don't have permission to access this resource. Either the resource belongs to a different organization, or your role on the organization does not grant the required permission. See [authorization errors](/api-reference/overview#authorization-errors) for the response body. |
