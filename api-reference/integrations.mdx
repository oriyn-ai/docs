---
title: "Integrations API endpoints"
description: "Integration providers and connections are managed through the Oriyn web app, not the REST API."
---

Integrations connect Oriyn to external data sources such as analytics platforms. Connection management — listing providers, viewing active connections, and deleting connections — is handled entirely through the [Oriyn web app](https://app.oriyn.ai). The REST API does not currently expose endpoints for managing integration providers or connections.

<Warning>
The previously documented `GET /integrations/providers`, `GET /integrations/connections`, and `DELETE /integrations/connections/{connection_id}` endpoints are deprecated and not available in the current `/v1` API surface. CRUD over integrations stays in the web app.
</Warning>

## Connecting a data source

To connect a provider such as PostHog:

1. Sign in at [app.oriyn.ai](https://app.oriyn.ai).
2. Open **Settings → Integrations**.
3. Choose your provider, complete the OAuth flow, and authorize access.

Once connected, the integration immediately becomes available for enrichment runs on any product in your organization.

## Removing a data source

To disconnect a provider:

1. Open **Settings → Integrations** in the [web app](https://app.oriyn.ai).
2. Select the connection you want to remove and confirm the deletion.

Disconnecting does not affect data already ingested into Oriyn — only future ingestion is stopped.

## Programmatic access

If your workflow requires programmatic discovery of available providers, the unauthenticated discovery endpoint at [`GET /v1/`](/api-reference/overview#discovering-the-api) returns the list of resource families exposed by the API along with a link to the full OpenAPI spec.

For PostHog-specific guidance, see the [PostHog integration guide](/integrations/posthog).
