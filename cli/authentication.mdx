---
title: "Authentication"
description: "Authenticate the Oriyn CLI using a browser-based OAuth flow. Credentials are stored securely in your OS keychain."
---

The Oriyn CLI authenticates you through a browser-based OAuth flow. After you complete login, credentials are saved to your operating system's keychain — not to a file on disk or an environment variable.

## Log in

<Steps>
  <Step title="Run oriyn login">
    Start the authentication flow from your terminal:

    ```bash
    oriyn login
    ```
  </Step>

  <Step title="Complete login in your browser">
    Your browser opens to `app.oriyn.ai`. Log in with your existing Oriyn account or sign up for a new one. If your browser does not open automatically, the CLI prints a URL you can copy and paste manually.
  </Step>

  <Step title="Return to your terminal">
    After you complete the login flow in the browser, return to your terminal. The CLI confirms that your credentials have been saved.
  </Step>

  <Step title="Verify your session">
    Confirm you are authenticated by running:

    ```bash
    oriyn whoami
    ```

    This prints your authenticated user's **email address** and **user ID**.
  </Step>
</Steps>

## Check your current user

At any time, you can see who is currently authenticated:

```bash
oriyn whoami
```

Example output:

```
Email:   you@example.com
User ID: usr_01abc123def456
```

<Tip>
  Pass `--json` to get machine-readable output from `whoami`, which is useful in scripts or CI pipelines:

  ```bash
  oriyn whoami --json
  ```
</Tip>

## Log out

To remove your stored credentials from the OS keychain, run:

```bash
oriyn logout
```

After logging out, commands that require authentication will prompt you to run `oriyn login` again.

## Security

Your credentials are stored exclusively in your operating system's keychain (Keychain on macOS, Secret Service on Linux, Windows Credential Manager on Windows). The CLI never writes tokens to plain text files, shell profiles, or environment variables.
